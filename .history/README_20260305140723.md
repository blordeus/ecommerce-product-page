# E-commerce Product Page (Frontend Mentor)

Responsive e-commerce product page built with React. Includes a product image gallery with lightbox, quantity selection, and a cart that persists via localStorage.

## Live Demo
https://blordeus.github.io/ecommerce-product-page

## Features
- Responsive layout (mobile → desktop)
- Product gallery with thumbnail navigation
- Lightbox gallery (opens from the main product image)
- Quantity selector + Add to Cart
- Cart modal with item removal and total
- Cart persisted in localStorage
- Mobile hamburger menu with overlay

## Tech Stack
- React (Create React App)
- Vanilla CSS
- react-icons
- GitHub Pages (deployment)

## Architecture / Component Structure

The application is organized into reusable React components that manage specific parts of the UI.

src
│
├── components
│   ├── Header.js
│   ├── CartModal.js
│   ├── ProductGallery.js
│   ├── QuantitySelector.js
│   └── MobileMenu.js
│
├── App.js
└── index.js

### Key Components

**App**
- Manages global cart state
- Handles add/remove cart logic
- Passes state and handlers to child components

**ProductGallery**
- Displays product images
- Supports thumbnail switching
- Opens lightbox gallery on larger screens

**CartModal**
- Displays cart items
- Allows item removal
- Shows calculated total

**QuantitySelector**
- Handles product quantity before adding to cart

**Header / MobileMenu**
- Navigation UI
- Mobile hamburger menu with overlay

## Data Flow

Cart state is managed in the `App` component and passed down to child components through props.

1. **User selects quantity**  
   `QuantitySelector` updates the selected quantity locally.

2. **User clicks "Add to Cart"**  
   The quantity and product details are passed to a handler in `App`.

3. **Cart state updates**  
   `App` updates the cart items and total.

4. **Cart persists**  
   The cart is saved to `localStorage` so it remains after page refresh.

5. **Cart UI updates**  
   `CartModal` receives the updated cart data and renders the items and total.
